title: "Rendering & Export"
description: "Exporting your emails to HTML, JSON, and React components"
---

Open Email Editor separates the **document model** (the JSON structure) from the **renderer** (the visual output). This allows you to export your emails in multiple formats.

## JSON Format

The editor uses a custom JSON schema to represent the email structure. This is the source of truth for the document.

### Exporting to JSON

```tsx
import { exportToJSON, useEditor } from "@open-email/editor";

function SaveButton() {
  const { document } = useEditor();

  const handleSave = () => {
    const json = exportToJSON(document);
    console.log(json);
  };

  return <button onClick={handleSave}>Save JSON</button>;
}
```

### Importing from JSON

```tsx
import { importFromJSON } from "@open-email/editor";

const jsonString = "{...}"; // Your saved JSON
const doc = importFromJSON(jsonString);
```

## HTML Export

You can render the current document state directly to a static HTML string, suitable for sending via any email service provider (ESP).

```tsx
import { renderToHTML } from "@open-email/editor";

const html = await renderToHTML(document);
```

### Server-side rendering (Next.js, Node.js)

The main package entry is built with `"use client"` for the editor UI. To call `renderToHTML` (or any other utility) from **server code**—e.g. Next.js API routes, Server Actions, or Route Handlers—use the **server** entry point:

```tsx
// API route, Server Action, or server-only module
import {
  renderToHTML,
  renderToPlainText,
  renderToReactEmail,
  exportToJSON,
  importFromJSON,
  interpolateVariables,
  createNode,
  createEmptyDocument,
  defaultRegistry,
  getAISchema,
} from "@open-email/editor/server";

const html = await renderToHTML(document, variableData);
```

Everything in `@open-email/editor/server` is safe to import in Node.js, Edge Runtime, and Next.js Server Components. Use `@open-email/editor` (without `/server`) only in client components that render the visual editor UI.

### HTML Export with Variables

When your document contains variables (`{{variableName}}`), pass `variableData` to interpolate them:

```tsx
const html = await renderToHTML(document, {
  firstName: "John",
  companyName: "Acme Corp"
});
```

Variables will be replaced with values from `variableData`, falling back to document-level variable definitions if not provided.

## React Email Compatibility

Under the hood, Open Email uses [React Email](https://react.email) components. The `renderToReactEmail` function converts the internal document model into a React Element tree composed of standard React Email components (`<Html>`, `<Section>`, `<Text>`, etc.).

```tsx
import { renderToReactEmail } from "@open-email/editor";
import { render } from "@react-email/render";

// 1. Convert to React Element
const reactElement = renderToReactEmail(document);

// 2. Render to HTML using React Email's renderer (optional, if you want full control)
const html = render(reactElement);
```
